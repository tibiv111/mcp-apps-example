R Shiny — integration options

Embedding R Shiny inside an MCP App is not a single decision. Four approaches are shown below, each pointing at the same Shiny service ({{ shiny_url }}). Status badges report what we've empirically observed in browser preview (loading /ui/shell directly) versus inside Claude's MCP host, which enforces frame-src 'self' blob: data: and a sandbox without allow-popups.

Diagnosis In Claude, the shell HTML is rendered under an opaque host origin, so 'self' in frame-src 'self' is not our service URL. That blocks every embed from inside the shell — even back to our own domain. Card D (URL-form resource) is the only spec-correct path; it asks the host to open its own iframe with its own permissions.

B · OPEN IN NEW TAB

Punch out of the iframe sandbox entirely

✓ BROWSER PREVIEW ✗ BLOCKED IN CLAUDE

window.open(SHINY_URL, '_blank'). In Claude, fails with Blocked opening … in a new window because the request was made in a sandboxed frame whose 'allow-popups' permission is not set — the parent (host-owned) sandbox is missing allow-popups; nothing the resource HTML sets can override it.

target: {{ shiny_url }}
C · SAME-ORIGIN REVERSE PROXY

Front Shiny with this service so the iframe is same-origin

✓ BROWSER PREVIEW ✗ BLOCKED IN CLAUDE

FastAPI proxies /shiny-proxy/* (HTTP + WebSocket) to the Shiny service so the iframe target is on our domain — see app/shiny_proxy.py. In a normal browser this passes frame-src 'self'. In Claude it still fails: the shell document is loaded under the host's opaque origin, so even our own service URL counts as cross-origin from inside the shell.

D · URL-FORM MCP RESOURCE

Let the host open its own iframe

✓ SPEC-CORRECT ? DEPENDS ON HOST

The MCP server returns a URL-form resource ({{ '"' }}mimeType{{ '"' }}: {{ '"' }}text/uri-list;profile=mcp-app{{ '"' }}) with _meta.ui.externalUrl set to {{ shiny_url }}. A compliant host fetches this resource and opens its own iframe at the URL — the shell's CSP is never in the picture. Whether Claude renders URL-form resources today is an empirical question; the button below issues a real tools/call launch_shiny and you can watch the host's response in /diagnostics.
