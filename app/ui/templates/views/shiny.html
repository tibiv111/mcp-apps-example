Expose Shiny as its own MCP server, peer to NAV AI

✓ ARCHITECTURALLY CLEAN ✓ MOUNTS IN CLAUDE

Per the MCP App spec the host mounts a fresh iframe per tool call, with multiple iframes coexisting in one session — so a peer iframe appears at the assistant turn that ran launch_shiny_dashboard. Visibility is gated on ui/notifications/initialized coming back from the iframe; we inline a minimal MCP App handshake (static/mcp-app-handshake.js) into the embedded HTML so the iframe announces itself before any of Shiny's own scripts run.

MCP server URL {{ base_url }}/shiny-mcp
Tools launch_shiny_dashboard · launch_shiny_hello
Resources ui://shiny/dashboard · ui://shiny/hello
MIME text/html;profile=mcp-app

To try it:

  1. Add {{ base_url }}/shiny-mcp as a connector in Claude (separate from the existing NAV AI server).
  2. In chat, ask Claude: "open the Shiny dashboard". A peer iframe should appear alongside the NAV AI workspace.
  3. To run the bi-directional demo end-to-end inside Claude, open the instead — it ships the working Shiny embed plus the hub-side controls.
